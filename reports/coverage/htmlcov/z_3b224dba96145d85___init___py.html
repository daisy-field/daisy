6"""Collection of executable scripts to start up the various components of the daisy 

7package (as a framework), more or less pre-configured by the respective script for 

8ease of use. Some of the scripts and entrypoints inside the respective subpackages 

9function as generic components (see below) served by the framework, while other are 

10more narrow configurations/implementations of said components to be run a demo 

11examples to work as a functioning setup. Most likely, for a full demo, one requires 

12scripts from different packages, as demo setups may require generic components as 

13well. See the docstrings of the respective demos. No matter the subpackage, 

14the scripts can be started either in separate python instances or via threads. For 

15convenience, one can also launch these scripts directly via the command line. 

16 

17Currently, the following (sub-)packaged scripts are provided: 

18 

19 * data_collection - Pre-configured scripts to label, process and store monitoring 

20 data from pre-selected typs of data sources. 

21 * demo_components - Pre-configured demo components for specific inputs/scenarios. 

22 * generic_fids_components - General federated IDS components required for
